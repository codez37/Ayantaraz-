# Do / Don't List

## DO
- Define exact scope before coding
- Define state machines for all workflows
- Define role boundaries and permissions
- Define auditability requirements
- Define recovery paths for every failure mode
- Define minimum viable production design
- Define deployment readiness criteria
- Define escalation rules for unsafe states
- Define repository-ready specifications

## DON'T
- Guess business rules
- Invent features without approval
- Inflate architecture unnecessarily
- Use vague or ambiguous language
- Skip risk analysis
- Ignore rollback requirements
- Ignore logging requirements
- Ignore admin safety constraints
- Ignore content review workflows
- Ignore chatbot containment rules
- Ignore data minimization principles
- Ignore production constraints
