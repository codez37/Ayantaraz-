# Payment Reference Policy

## Definition
A payment reference is the admin's recorded evidence of payment completion. It is NOT a gateway receipt but a manual trace of what was received.

## Acceptable References
- Bank transfer receipt number (شماره پیگیری واریز)
- Cash payment confirmation note
- Check number
- Internal transaction ID
- Admin note confirming verbal payment confirmation

## Recording Rules
- Reference must be at least 5 characters
- Reference should include the method (card/cash/check/transfer)
- Timestamp is auto-recorded
- Admin who recorded it is logged in audit

## Prohibited
- Empty or "confirmed" without reference
- Reference without admin identity
- Confirmation before actual payment received
